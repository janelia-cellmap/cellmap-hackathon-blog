Metadata discussion

John Bogovic presents metadata standards and translation with jq

Jun 10, 2022 • 2 min read

metadata   json   jq   zarr   ome-ngff   transformations

John Bogovic (@bogovicj) from the Saalfeld lab at Janelia gave a demonstration and led a discussion on metadata related topics.

“Translation” demo and tutorial

Metadata are important in correctly interpreting and processing data, but software often require that these metadata be formatted in a particular way. Software differ in exactly what the particular way should be.

Metadata “translation” takes metadata formatted in one way and produces the same metadata formatted in for another. We have found that doing this on-the-fly (without modifying files) is useful, especially when modifying the source metadata is not possible or practical.

The demo included:

  • A description and demo of the JSON processor jq.
  • Working through this tutorial for converting between two different N5 metadata “dialects”.
    • Pointers to existing jq helper functions for n5.
    • See below for the example script we learned from.
  • A demonstration of on-the-fly metadata conversion built into Fiji’s N5 plugin.
    • Virtually modify the names / paths of datasets using the n5 container tree.

NGFF specification discussion

Next, the group discussed the coordinate transformation metadata standard that John is preparing for OME’s “Next generation file formats (NGFF)” specification.

Specifically discussed were:

  • Current and future use cases for the specification.
  • The benefits of defining coordinate systems (“spaces”).
  • What types of transformations should be included?
  • Describing transformations that apply to a subset of coordinates
    • What name the spec should give to particular transformations1.
    • How much is appropriately left implicit for convenience.
    • Thinking about points as lists or with structure.

More details about OME-NGFF can be found in the paper, or in the specification. As well, there is a brief discussion of OME-NGFF in John Bogovic’s EMBL-Janelia seminar.

jq tutorial script

Given the n5.jq helper functions (see arrayAndUnitToTransform), the script:

include "n5";
def isN5v: type == "object" and has("pixelResolution");

def convert: .pixelResolution |
    [ [ .dimensions[0], 0, 0, 0,
        0, .dimensions[1], 0, 0,
        0, 0, .dimensions[2], 0 ],
      .unit ]
    | arrayAndUnitToTransform;

walk( if isN5v then . + convert else . end )

takes this input:

{
    "pixelResolution": {
        "dimensions": [8, 7, 6],
        "unit": "mm"
    }
}

and produces this output:

{
    "pixelResolution": {
        "dimensions": [8, 7, 6],
        "unit": "mm"
    },
    "spatialTransform": {
        "transform":{
            "type": "affine",
            "affine": [8, 0, 0, 0, 0, 7, 0, 0, 0, 0, 6, 0]
        },
        "unit":"mm"
    }
}
